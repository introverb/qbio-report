## 1. Project summary

**QUBIE Report** (a.k.a. QUBIE News) is a Flask news aggregator for the
**Quantum Biology DAO**. A Python scraper pulls articles from RSS feeds,
search APIs (PubMed, arXiv, Europe PMC, Semantic Scholar, OpenAlex), forums
(Reddit, Hacker News, Stack Exchange), social (Bluesky), and YouTube; scores
them by keyword match against `keywords.txt`; deduplicates; runs a Haiku
relevance pass on chatter/social/video to drop coincidental keyword matches;
then writes `feed.json`. The static site (Flask serves the HTML) renders the
feed across `/`, `/chatter`, `/video`, with auth-gated personal saves
(SQLite-backed) and an `/admin` console for source/keyword/filter management.

**Stack:** Python 3.11, Flask + APScheduler (twice-daily cron at 07:00 and
16:00 PT), SQLite for users + saves, Anthropic Claude Haiku 4.5 for blurbs
and the relevance filter, gunicorn under Railway with a persistent volume
mounted at `/data`.

**Deployed at:** https://qubiereport.up.railway.app/ — Railway service `web`
in project `marvelous-heart`. GitHub repo: `introverb/qbio-report` (public).

### Major features

- **Scraper** (`scraper/scraper.py`) writes `feed.json`, `sources.json`,
  `QBIO-Report-Sources.xlsx`, `scrape_progress.json`, `rejected.json` to
  `DATA_DIR` (= `/data` on Railway, repo root locally).
- **Real progress bar** on `/admin` polls `/api/scrape-progress` every 1.5s
  during a scrape — pink fill, shows `done/total · N% · now: <source>`.
- **Auth**: SQLite at `qubie.db` on the volume; bcrypt password hashing;
  cookie sessions (Flask `secret_key`); bootstrap admin via env var
  `BOOTSTRAP_ADMIN_USERNAME` (matches lowercased username; auto-promote runs
  on every authenticated request, not just signup).
- **Personal saves**: bookmark icon on every feed item, pink linework, fills
  pink when saved. Logged-out users get redirected to `/login` on click.
  Save data lives in SQLite `saves` table; survives `feed.json` regeneration

- **"Saved by the DAO"** on the homepage — when search is empty, shows top
  20 saves (last 30 days) with `Latest save: @username` attribution. Typing
  flips the section to "Search the Feed".
- **Manage Users** on `/admin` — table of all accounts with Make-admin /
  Remove-admin buttons. Use this to promote Kyra (or anyone else) once they
  sign up.
- **Filtered Items** on `/admin` — items dropped by the relevance filter
  with Restore (whitelists + re-adds to feed.json) and Dismiss buttons.
- **Times** all render in Pacific Time (`America/Los_Angeles`) site-wide.
  Scrapes fire at 07:00 and 16:00 PT (configurable via `SCRAPE_HOURS` env
  var).

## 6. Data / config notes

### Files on the volume (`/data` on Railway, repo root locally)

| File | Purpose | Written by |
| --- | --- | --- |
| `feed.json` | Main article feed; rendered by all feed pages | scraper |
| `sources.json` | Per-source live stats (matched/total/error) | scraper |
| `QBIO-Report-Sources.xlsx` | Same data + Source Requests + Keyword Suggestions tabs | scraper + admin POSTs |
| `scrape_progress.json` | Live scrape progress (done/total/current) | scraper |
| `rejected.json` | Items the relevance filter dropped (60-day TTL) | scraper |
| `whitelist.json` | Links the admin Restore'd; bypass filter forever | server (Restore action) |
| `keywords.txt` | One keyword per line; `keyword [N]` for weight N | bundled, then user edits |
| `blurbs_cache.json` | Article-link → Haiku blurb cache | scraper (in `enrich_with_blurbs`) |
| `qubie.db` | SQLite: users + saves tables | server |
| `sources_config.json` | Runtime-pushed RSS feeds + subreddits (admin "Push" button) | server |
| `source_requests_log.md` | Newest-first append log of public source suggestions | server |

### How the scraper runs

Twice-daily cron at **07:00 and 16:00 PT** (configurable via `SCRAPE_HOURS`,
default `7,16`; APScheduler timezone is `America/Los_Angeles`). Manual
triggers via `/admin → Run scrape now` (POST to `/api/scrape`, runs in a
background thread). Server invokes `python scraper/scraper.py` as a
subprocess, 20-minute timeout cap.

Scraper main flow:
1. Load `keywords.txt`, merge runtime sources from `sources_config.json`
2. Run all tiers, append to `all_articles[]`, `progress_tick()` per source
3. Dedupe by lowercase title, sort by date desc
4. **`filter_off_topic()`** — Haiku relevance pass on
   chatter/social/video (skips whitelisted links, batches 20 per Haiku call)
5. **`enrich_with_blurbs()`** — Haiku blurb generation for top-N articles
   (cached aggressively; re-runs are cheap)
6. Write `feed.json` + `sources.json` + xlsx + `progress_finish()`

### Where Anthropic API is called

Two places in `scraper/scraper.py`:
- `_judge_batch(client, batch)` — relevance filter (model `claude-haiku-4-5`,
  ~12 tokens/article output, batched 20-per-call)
- `_generate_one_blurb(client, article)` — blurb generation
  (model `claude-haiku-4-5`, max 150 tokens)

Both read `os.environ["ANTHROPIC_API_KEY"]`. If missing, they print a
"skipping" notice and exit without erroring (fail-open).

### Keyword weighting

`keywords.txt` lines like `quantum biology [5]` mean weight 5. Default is 1.
`load_keywords()` returns a `{phrase: weight}` dict; `score_article()` sums
matched weights. Bracket syntax is parsed by both the scraper and the admin
UI's regex (`_KW_WEIGHT_RE`).

## 7. Gotchas

### Env vars (set in Railway → web → Variables)

Required:
- `ANTHROPIC_API_KEY` — Claude Haiku for blurbs + filter
- `ADMIN_PASSWORD` — currently `microtubular` per Olli's pick
  (Penrose–Hameroff microtubule nod); used for the legacy login fallback
- `FLASK_SECRET_KEY` — must be set in production; rotating invalidates all
  sessions
- `DATA_DIR` — `/data` on Railway (volume mount)

For specific tiers:
- `YOUTUBE_API_KEY` — Tier 4. Free tier = 10k quota/day; one search.list call
  costs 100 units. We chunk keywords into ~5 calls/scrape = well under cap.
- `REDDIT_CLIENT_ID` / `REDDIT_CLIENT_SECRET` — Tier 3a. Required because
  Reddit blocks generic User-Agents and cloud-provider IPs without OAuth.
- `SEMANTIC_SCHOLAR_API_KEY` — optional; without it, Tier 2 skips Semantic
  Scholar (their public endpoint heavily rate-limits cloud IPs).

Auth:
- `BOOTSTRAP_ADMIN_USERNAME` — currently `op1f.jpg`. First user matching
  this gets `is_admin=1` on signup OR on next request after setting it
  (auto-promote runs in `_current_user()`).

Optional:
- `SCRAPE_HOURS` — comma-separated PT hours (default `7,16`)
- `SCRAPE_ON_STARTUP` — set to `1` to fire a warm-up scrape ~60s after boot
- `PORT` — Railway sets this automatically

### Lock-out protection (active)

Legacy admin login is intentionally still active. To use: visit `/login`,
**leave the username field blank**, type the `ADMIN_PASSWORD`. Sets
`session["authed"] = True`. `_current_user()` returns a synthesized
`_legacy_admin` dict that's treated as admin everywhere. Once Kyra is set up,
remove this code path (see open items).

### Cache-busting

`assets/saves.js` and `assets/user-nav.js` are loaded with `?v=4` query
strings on every page that uses them. **Bump the version when you change
either file** or browsers will serve stale copies after deploy. Search the
codebase for `?v=4` to find every call site.
